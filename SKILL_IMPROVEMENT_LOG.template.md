# Skill / Kit Improvement Log

Copy file này thành `<theme>/deliverables/SKILL_IMPROVEMENT_LOG.md` khi bắt đầu dự án. Mục đích: gom mọi tín hiệu nâng cấp kit/skill từ dự án thực tế để dự án sau tốt hơn. Kit là READ-ONLY khi chạy project — log ở đây, KHÔNG sửa kit giữa chừng.

## Cách ghi

Append NGAY KHI phát sinh (không đợi cuối phiên), mỗi entry một dòng:

```
- [tag] YYYY-MM-DD — chuyện gì xảy ra → đề xuất sửa kit/skill
```

| Tag | Khi nào |
|---|---|
| `kit-bug` | Script/template/doc của kit sai hoặc gãy |
| `cli-gap` | Haravan CLI thiếu khả năng, phải làm thủ công |
| `skill-gap` | Rule skill mù mờ / thiếu / mâu thuẫn khi áp vào thực tế |
| `user-request` | BẤT KỲ yêu cầu thay đổi/feedback nào từ user trong dự án |
| `workaround` | Cách lách đã dùng (kèm lý do) — ứng viên chuẩn hóa |
| `new-pitfall` | Bẫy mới phát hiện (platform/theme/tool) đáng đưa vào docs |

## Nghi thức

- Surface các entry MỚI ở mỗi STOP (1/2/3); toàn bộ list vào evidence pack STOP 3.
- Kết thúc dự án: review log, entry đáng giá → phiên maintenance kit riêng (có approval) để cập nhật kit + CHANGELOG + skill 3 bản (Claude/Copilot/Codex).
